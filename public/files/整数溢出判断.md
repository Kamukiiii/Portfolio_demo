## 📚考点分析：**整数溢出判断**

### 1. 符号位判断法（最直观的方法）

这种方法是通过分析**运算数**和**结果**的符号来判断是否发生溢出。溢出只可能发生在**两个同符号数相加**（或两个异符号数相减，因为减法转换为加法）。

| **运算类型**    | **溢出发生的条件**    | **溢出结果的符号**                   |
| --------------- | --------------------- | ------------------------------------ |
| **正数 + 正数** | 结果符号为 **负**     | 结果溢出到负数的范围（向上溢出：上溢 |
| **负数 + 负数** | 结果符号为 **正**     | 结果溢出到正数的范围（向下溢出：下溢 |
| **异号数相加**  | **绝不会** 发生溢出。 | -                                    |

**总结：** 当两个同符号数相加，而结果的符号与加数的符号**相反**时，即发生了溢出。

### **4.知识点总结（在 C 语言中的体现）**

在 C 语言中，溢出处理机制与计算机硬件紧密相关：

1. 有符号整数（Signed Integer）溢出：
   1. C 语言标准规定有符号整数溢出是未定义行为（Undefined Behavior）。实际在多数现代计算机上，它会像硬件一样进行补码运算并发生“回绕”（Wrap Around），但由于是未定义行为，我们通常需要通过上述方法来提前检测或使用更大的数据类型来避免。
2. 无符号整数（Unsigned Integer）溢出：
   1. C 语言标准规定无符号整数运算是模运算（Modulo Arithmetic）。例如，一个 32 位的 `unsigned int` 溢出后，结果是 。这在技术上不叫错误溢出，因为它有明确的定义。

> 如果是补码的减法运算，那么溢出判断则根据减数取反、减法变成加法之后的两个加数的符号进行判断
